# Predictive Sales Forecasting & Retail Analytics Engine

![Cover Photo](charts/anomaly_isolation_forest.png)

## Overview

This repository showcases the implementation of an end-to-end machine learning and analytics pipeline designed for retail sales forecasting and inventory optimization. It is designed to explore how predictive models can guide business decisions through data analysis, forecasted demand, and anomaly detection.

Currently, the project focuses on predicting future sales and segmenting products, providing a solid foundation for supply chain management and retail analytics.

- **Objective:** Solve the overstocking and understocking problem by accurately predicting future product demand.
- **Approach:** Moves beyond historical averages by utilizing statistical and ML-based forecasting, isolation forests, and clustering algorithms.
- **Output:** A sleek, interactive Streamlit dashboard designed specifically for business executives to make data-driven decisions.

### 2. Anomaly Detection

An automated monitoring system designed to flag unusual spikes or drops in weekly sales data.

<details>
<summary><b>View Detailed Specifications</b></summary>

- **Objective:** Instantly flag abnormal weeks to allow management to investigate supply chain failures or unexpected viral trends.

### 3. Product Segmentation

A clustering engine that profiles products based on their demand behavior to optimize warehousing strategies.

<details>

## Getting Started

### Prerequisites

Before you begin, ensure you have the following installed:

1. **Python 3.10+** (Required for the backend data processing).
2. **Astral uv** (Recommended package manager for extremely fast environment setup).
3. **Git** (For version control).

### Installation & Setup

1. **Clone the repository:**

   ```bash
   git clone <your-repo-url>
   cd SalesForecasting
   ```

2. **Set up the Python Environment:**
   It is highly recommended to use `uv` to manage the virtual environment and install dependencies cleanly.

   ```bash
   # Install dependencies from the requirements file
   uv pip install -r requirements.txt
   ```

3. **Generate the Dashboard Data:**
   Before running the dashboard, the raw data must be processed and serialized.
   - Open `analysis.ipynb` in your preferred Jupyter environment.
   - Run all the cells to train the models and export the required DataFrames.
   - The final cell will generate a `dashboard_data.pkl` file in the root directory.

---

## Running the Dashboard

To initiate the interactive web application:

1. Open your terminal and ensure your virtual environment is active.
2. Navigate to the root directory of the project.
3. Run the Streamlit CLI command using `uv`:
   ```bash
   uv run streamlit run app.py
   ```
4. The dashboard will automatically launch in your default web browser (typically at `http://localhost:8501`).
